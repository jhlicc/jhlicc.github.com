Q: 17.4 为什么一些人写 if(0==x) 而不是 if(x==0) ？

A: 它是一个用来防止这种常见错误的小把戏 

  if (x = 0)

如果你习惯把常量写在==之前，编译器会报错，如果你偶尔这样写 

  if (0 = x)

显然对一些人来说，记住倒转测试比记住键入两个=符号更容易。（固然偶尔写成=而不是==，是一个甚至最有经验的程序员也会犯的打字错误。）

另一方面，一些人发现这些倒转的测试很丑陋或混淆，并且争辩编译器应该对 if(x = 0) 发出警告。（实际上许多编译器确实对条件中的赋值产生警告，然而你总是可以写 if((x = expression)) 或 if((x = expression) != 0) 如果你真的是这个意思。）

参考：H&S Sec. 7.6.5 pp. 209-10
